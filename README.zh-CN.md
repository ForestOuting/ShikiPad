# ShikiPad — Windows 手柄键鼠映射工具

> 将 **DualSense (PS5)** · **DualShock 4 (PS4)** · **Xbox 360** · **Xbox Series X|S** 手柄映射为键盘和鼠标。

ShikiPad 是一款为 **"用手柄高效打字"** 而深度优化的 Windows 原生应用程序。无论你是躺在床上操作电脑、使用掌上串流设备，还是需要在无键盘环境下进行文字输入，ShikiPad 都能提供流畅、精准的字符输入和鼠标操控体验。

核心特性：
- 🎯 **26 个英文字母 + 数字 + 标点**一键可达，按键布局按照字母使用频率精心排列
- 🖱️ 右摇杆控制鼠标光标，支持可调节的加速曲线和死区
- ⚡ 独创"蓄力 (Clutch)"系统，轻松输入 `Ctrl + Shift + Esc` 等多重修饰键组合
- 🛡️ 基于 Interception 内核驱动，可在虚拟机和部分反作弊游戏中正常工作
- ⚙️ 所有运行时参数均可通过 JSON 配置文件微调

## 📥 安装与运行

### 1. 下载
在 GitHub 页面右侧的 **Releases** 栏下载最新版压缩包（或直接 Clone 本仓库），解压至任意目录。

### 2. 安装底层驱动（必须）
ShikiPad 使用 [Interception](https://github.com/oblitum/Interception) 内核驱动来实现硬件级键盘注入，确保在虚拟机和带有底层反作弊的游戏中也能正常工作。

1. 在解压目录中找到 **`install_driver.bat`**。
2. **右键 → 以管理员身份运行**。
3. 等待命令行窗口提示 `Installation complete!` 后关闭窗口。
4. ⚠️ **必须重启电脑**，驱动才能生效。

### 3. 启动程序
重启后双击 `ShikiPad.exe`，在弹出的终端窗口中选择你的手柄型号 `[1..8, Enter = 1]`：

| 编号 | 手柄型号 | 连接方式 |
|:---:|---|---|
| 1 | DualSense (PS5) | USB |
| 2 | DualSense (PS5) | 蓝牙 |
| 3 | DualShock 4 (PS4) | USB |
| 4 | DualShock 4 (PS4) | 蓝牙 |
| 5 | Xbox 360 | USB |
| 6 | Xbox 360 | 蓝牙 |
| 7 | Xbox Series X\|S | USB |
| 8 | Xbox Series X\|S | 蓝牙 |

选择后程序自动进入后台运行。**关闭终端窗口即可退出程序。**

### 4. 紧急开关
在运行过程中，**同时按住分享键 (Create/View) + 设置键 (Options/Menu) 持续 2 秒**，即可切换启用/禁用状态——当你需要暂时恢复手柄的原始输入时，这非常有用。

## 🎮 完整操控指南

### 🖱️ 鼠标操作（右摇杆）

| 手柄输入 | 电脑输出 | 说明 |
|---|---|---|
| **右摇杆** 移动 | 鼠标光标移动 | 支持幂函数加速曲线，轻推精准、重推迅速 |
| 按下 **R3**（右摇杆按钮） | 鼠标右键 | 按下后会短暂冻结光标，防止点击时产生位移 |
| 按下 **L3**（左摇杆按钮） | 鼠标左键 | — |
| **PS 分享键 / Create** (DS4/DS5) | `Right Alt` | 按住即按住，松开即释放 |
| **PS 设置键 / Options** (DS4/DS5) | `Right Ctrl` | 按住即按住，松开即释放 |
| **PS Home 键** (DS4/DS5) | `Right Shift` | 按住即按住，松开即释放。Xbox 手柄的 Home 键被 Windows 系统拦截用于 Xbox Game Bar，无法映射 |

### 🛠️ 修饰键与功能键（左摇杆）

左摇杆充当一个 **8 方向功能盘**：将摇杆推向某个方向即按住对应键，松开回到中心即释放。

| 方向 | 输出 | 典型用途 |
|---|---|---|
| **← 左** | `Shift` | 输入大写字母、上标符号 |
| **↙ 左下** | `Ctrl` | 快捷键（如 Ctrl+C 复制） |
| **↘ 右下** | `Left Alt` | 快捷键（如 Alt+Tab 切窗口） |
| **→ 右** | `Win` | 打开开始菜单、系统快捷键 |
| **↖ 左上** | `Esc` | 退出、取消操作 |
| **↑ 上** | 鼠标滚轮↑ | 网页/文档向上滚动（长推加速） |
| **↓ 下** | 鼠标滚轮↓ | 网页/文档向下滚动（长推加速） |
| **↗ 右上** | 激活 `Fn` 层 | 配合数字键输出 F1~F12 |

#### ⚡ 蓄力系统（Clutch —— 修饰键收集器）

在默认状态下，左摇杆同一时间只能按住一个方向的修饰键。当你需要输入像 `Ctrl + Shift + Esc`（任务管理器）这样的多重修饰键组合时，就需要使用**蓄力**功能：

**触发方式：**

| 手柄类型 | 开启蓄力 | 解除蓄力 |
|---|---|---|
| DS4 / DS5 | 短按 **触控板** 切换开启，或长按触控板按住蓄力 | 再短按一次触控板，或长按后松开 |
| Xbox | 短按 **View / Back** 或 **Menu / Start** 切换开启，或长按任一按钮按住蓄力 | 再短按一次蓄力按钮，或长按后松开 |

- **短按** = 点击切换蓄力开启 / 解除。
- **长按** = 按住多久就蓄力多久；松开按钮一定解除蓄力，即使之前已经通过短按开启了蓄力。
- 在 PS 手柄上，**分享键 / Create**、**设置键 / Options**、**Home 键**现在是空闲的实体键输出：`Right Alt`、`Right Ctrl`、`Right Shift`。

**使用方法：** 在蓄力期间，随意拨动左摇杆即可依次"收集"多个修饰键。例如先推向左（收集 `Shift`），再推向左下（收集 `Ctrl`），再推向左上（收集 `Esc`），三个键会同时保持按下——等效于键盘上同时按住 `Ctrl + Shift + Esc`！解除蓄力后，所有收集到的修饰键会一并释放。

#### Fn 功能键层（F1 ~ F12）

将左摇杆推向**右上方 (↗)**，手柄上的数字键 `1`~`0` 及 `-`、`=` 将变为 `F1` ~ `F12`。在推住 Fn 方向的同时按下对应键即可触发。

### ⌨️ 打字与动作按键

#### 基础层（不按任何肩键/扳机时）

| 手柄按键 | 输出 | 说明 |
|---|---|---|
| **方向键 ↑↓←→** | 键盘方向键 | 按住可连发 |
| **□ 方块** (Xbox: X) | `Space` 空格 | 按住可连发 |
| **△ 三角** (Xbox: Y) | `Backspace` 退格 | 按住可连发 |
| **✕ 叉** (Xbox: A) | `Enter` 回车 | 按住可连发 |
| **○ 圆** (Xbox: B) | `Tab` 制表符 | 按住可连发 |

> 基础层的按键**按住会连发**——初始延迟 300ms 后开始加速重复，与真实键盘行为一致。

#### 字符层（按住肩键/扳机时 —— 快速打字）

按住 **R1、L1、R2、L2** 任意一个（或组合按住），方向键和动作键将变为英文字母、数字或标点符号。

为防止快速连打时误触，**字符层的所有按键均为"点按"模式**——按下的瞬间即完成一次完整的按下+松开操作，按住不会连发。字母按照英文使用频率排列：最常用的 8 个字母 (`i n e a o t h u`) 分配在 R1 层，次常用的分配在 L1 层，以此类推。

**完整字符映射表：**

*按键顺序（从左到右）：上, 右, 方块, 三角, 左, 下, 叉, 圆*

| 按住的肩键/扳机 | ↑ | → | □(X) | △(Y) | ← | ↓ | ✕(A) | ○(B) |
|---|:---:|:---:|:---:|:---:|:---:|:---:|:---:|:---:|
| **R1 (RB)** | i | n | e | a | o | t | h | u |
| **L1 (LB)** | s | r | d | g | l | c | y | z |
| **R2 (RT)** | m | w | j | x | q | f | p | b |
| **L2 (LT)** | k | v | 1 | 2 | 3 | 4 | 5 | 6 |
| **R1 + L1** | 7 | 8 | 9 | 0 | - | = | , | . |
| **R2 + L2** | `<` | `)` | `[` | `{` | `(` | `>` | `}` | `]` |
| **L1 + R2** | `` ` `` | `\` | `'` | `"` | `;` | `~` | `/` | `?` |
| **R1 + L2** | `!` | `@` | `#` | `$` | `%` | `^` | `&` | `*` |

> **大写字母与少数 Shift 标点：** 先将左摇杆推向**左 (Shift)**，保持不放的同时按下对应按键。常用 Shift 标点现在已经放进组合层直出；少数低频符号如 `_`、`+`、`:`、`|` 仍可用左摇杆 Shift 搭配 `-`、`=`、`;`、`\` 输入。

> **💡 语音输入提示：** 如果你觉得手柄键盘打字依然不够快，可以使用 `Win + H`（`左摇杆 →` + `R1 + □/X`）快速开启 **Windows 自带语音输入**（建议在 `Windows 设置` → `隐私和安全性` → `语音` 中开启"在线语音识别"）。
> 更强烈推荐配合 **Typeless**、**Whisper** 或 **闪电说** 等第三方智能语音输入软件使用，以获得远超系统自带的识别速度与准确率！如果使用 DualSense (PS5) 手柄，还可以在系统声音设置中将输入设备切换为手柄内置麦克风，贴近嘴巴说话识别效果会更棒！

### 🔧 键层判定机制（进阶理解）

ShikiPad 使用一套精密的时序判定系统来确保你"按下字母键"还是"按下基础键"的意图被准确识别：

> **注：组合层判定时间现已从 30ms 调整至 35ms。**


| 参数 | 默认值 | 作用 |
|---|---|---|
| `comboLayerWindowMs` | 35ms | R1+L1、R2+L2、L1+R2 或 R1+L2 两个键的按下时间差必须在此窗口内，才会被判定为组合层 |
| `actionLayerGraceMs` | 35ms | 前置确认窗口。按下动作键后，如果肩键/扳机在此窗口内发生变化，新键层仍可参与最终归属判定 |
| `actionLayerPostGraceMs` | 35ms | 后置归属窗口。从键层松开后开始计算，只覆盖旧键层松开到新键层按下之间的空白期；若后续新键层的前置判定覆盖成功，则仍按新键层执行 |
| `layerTakeoverWindowMs` | 30ms | 按住旧键层时的重叠接管窗口。最大允许接管时间是 30ms，但最终能否接管取决于独立的待定键层归属运算；它不计算松开后的空白期 |

简单来说：前置确认 **35ms**，后置归属 **35ms**，组合层配对 **35ms**，按住旧键层时的新层接管重叠窗口 **30ms**。如果某个动作键是在明确键层里按下并已经松开，后续快速切到新键层不会再回头改写这个已经完成的字符。

## ⚙️ 进阶配置

程序首次运行后，会在同目录下自动生成 `shikipad.json` 配置文件。使用任意文本编辑器（如记事本）打开修改，保存后重启 ShikiPad 即可生效。合法的用户自定义数值会被保留；只有缺失、废弃或非法的配置项才会回退到默认值。

可参考 `shikipad.example.json` 作为干净的默认配置模板。

### 鼠标相关

| 参数名 | 默认值 | 说明 |
|---|---|---|
| `mouseMaxSpeed` | `20.0` | 右摇杆推满时的鼠标最大速度单位。实际最大速度约为 `mouseMaxSpeed * 120 * mouseSensitivity` 像素/秒 |
| `mouseSensitivity` | `1.0` | 鼠标灵敏度的全局乘数倍率 |
| `rightStickDeadzone` | `0.03` | 右摇杆静止噪声死区。用于过滤硬件回中漂移，同时保留越过空闲噪声带后的轻推移动 |
| `rightStickCurveExponent` | `3.0` | 幂函数曲线指数。值越大，轻推时越精准，推满时速度不变 |
| `mouseScrollCurveExponent`| `3.0` | 左摇杆鼠标滚轮的分层速度精细度指数。值越大，轻推滚轮越精细 |
| `r3FreezeMs` | `60` | 按下 R3（右键点击）后冻结光标的毫秒数。因为按下摇杆时很容易产生微小的位移导致点击失误，此参数可在按下瞬间屏蔽移动，保证点击稳定 |

### 左摇杆 / 修饰键

| 参数名 | 默认值 | 说明 |
|---|---|---|
| `leftStickEnterDeadzone` | `0.35` | 左摇杆需要推到此比例才会触发方向判定 |
| `leftStickExitDeadzone` | `0.25` | 左摇杆回到此比例以下才会判定为回中 |

### 扳机 / 肩键

| 参数名 | 默认值 | 说明 |
|---|---|---|
| `triggerPressThreshold` | `0` | L2/R2 扳机被判定为"按下"的阈值。`0` 表示只要有任何正向模拟输入就算按下 |
| `triggerReleaseThreshold` | `0` | L2/R2 扳机被判定为"释放"的阈值。默认 `0` 表示完全回到 0 才释放 |

### 打字与键层

| 参数名 | 默认值 | 说明 |
|---|---|---|
| `comboLayerWindowMs` | `35` | R1+L1 / R2+L2 / L1+R2 / R1+L2 组合层的最大按下时间差（毫秒） |
| `actionLayerGraceMs` | `35` | 前置确认窗口（毫秒）。动作键按下后，新键层在此窗口内按下可覆盖该动作键 |
| `actionLayerPostGraceMs` | `35` | 后置归属窗口（毫秒）。从键层松开后开始，只覆盖旧键层到新键层之间的空白期 |
| `layerTakeoverWindowMs` | `30` | 按住旧键层时的接管窗口（毫秒）。30ms 是最大允许重叠时间，最终是否接管仍由待定键层归属运算决定；不计算松开后的空白期 |
| `actionLayerSwitchGuardMs` | `35` | 已发出字符的换层保护期（毫秒）。它独立于后置归属，用于在字符已经发送后，继续按住动作键并切换键层时抑制残留误触 |
| `clutchLongPressMs` | `250` | 区分蓄力短按与长按的按住时长（毫秒） |

### 连发 / 滚轮

| 参数名 | 默认值 | 说明 |
|---|---|---|
| `repeatDelayMs` | `300` | 基础层按住后开始连发前的初始延迟 |
| `repeatIntervalMs` | `32` | 基础层连发到最快速度时的间隔，接近键盘长按的高档速度 |
| `baseRepeatSlowIntervalMs` | `240` | 连发加速前的起始重复间隔 |
| `baseRepeatRampMs` | `2500` | 从慢速连发渐变到最高连发速度所需的时间 |
| `scrollSlowIntervalMs` | `180` | 滚轮慢速参考间隔（毫秒）。左摇杆滚轮和右摇杆鼠标一样基于真实连续摇杆输入：摇杆偏移会被累计为滚轮增量；“从 0 平滑起步”描述的是死区附近的速度曲线，不是假的离散档位 |
| `scrollFastIntervalMs` | `18` | 摇杆推满时的最快滚动间隔（毫秒） |

### 系统

| 参数名 | 默认值 | 说明 |
|---|---|---|
| `configVersion` | `8` | 配置文件结构版本标记。除非发布说明要求，否则保持此值不变 |
| `useInterception` | `true` | 是否使用 Interception 内核驱动注入。设为 `false` 将降级为 `SendInput` 模式 |
| `useScanCode` | `true` | 是否发送硬件扫描码（对部分游戏和虚拟机兼容性更好） |

## 💡 常见问题

### 双重输入问题
如果你在游戏中同时收到了手柄的原始信号和 ShikiPad 的键盘信号：

**Sony 手柄（DualSense / DualShock 4）：** 安装 [HidHide](https://github.com/nefarius/HidHide/releases)，将你的物理手柄设备隐藏，并将 `ShikiPad.exe` 的完整路径加入 HidHide 的应用程序白名单。

> ⚠️ **注意 — Xbox 手柄无法被 HidHide 隐藏。** Xbox 手柄使用 XInput API（位于 API 层），而 HidHide 仅作用于 HID 设备层，对 XInput 设备完全无效。
>
> 如果你需要隐藏 Xbox 协议的手柄，建议选用支持切换至 **PS / DirectInput 模式**的第三方手柄，例如 **盖世小鸡 (GameSir)**、**飞智 (Flydigi)**、**北通 (Betop)** 等。切换到 DirectInput 模式后，手柄会被系统识别为标准 HID 设备，即可被 HidHide 正常隐藏。
